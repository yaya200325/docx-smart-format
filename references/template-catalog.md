# Template Catalog

## 目标

该文件定义每种内置模板应如何调用 Word 格式能力。

重点覆盖：

- 分节
- 页眉页脚
- 页码
- 中文字体
- 英文字体
- 标题层级
- 正文缩进
- 行距与段距
- 图片
- 表格
- 图注表注
- 公式

## 通用硬规则

除非模板明确覆盖，否则默认遵循这些规则：

### 页面与排版

- 页面：`A4` 纵向
- 正文行距：`1.5`
- 图注默认在图下方
- 表注默认在表上方
- 独立公式默认居中
- 页码必须使用 Word 页码字段
- 公式不默认编号，是否编号由 LLM 按模板与上下文判断

### 字体（中英文混排必须分别设置）

- 中文字体：`宋体`（`run.eastAsiaFont`）
- 英文字体：`Times New Roman`（`run.asciiFont`）
- 中英文混排时**必须分别设置**两者，不要只设一个总字体

常见备选：

| 选项 | 推荐默认 | 常见备选 |
|------|---------|---------|
| 中文字体 | 宋体 | 黑体、仿宋、楷体 |
| 英文字体 | Times New Roman | Arial、Calibri、Helvetica |

### 字号

正文与同级元素：

| 块类型 | 默认字号 | OpenXML 半点值 |
|--------|---------|----------------|
| 正文段落（`body`） | 小四 12pt | `"24"` |
| 图注、表注 | 小四 12pt（可略小至 10.5pt） | `"24"` 或 `"21"` |
| 页眉、页脚 | 小四 12pt | `"24"` |
| 参考文献 | 小四 12pt（可略小至 10.5pt） | `"24"` 或 `"21"` |

标题层级：

| 层级 | 默认字号 | OpenXML 半点值 |
|------|---------|----------------|
| 一级标题 | 小二 18pt | `"36"` |
| 二级标题 | 三号 16pt | `"32"` |
| 三级标题 | 四号 14pt | `"28"` |

常见备选：五号 10.5pt（`"21"`）、小三 15pt（`"30"`）。

## 模板一：`academic-thesis`

适用：

- 本科论文
- 学位论文
- 学术研究型长文

### 分区总则

论文通常至少分成这些分区：

1. 封面
2. 摘要
3. 目录
4. 正文
5. 参考文献
6. 附录

这些分区不一定每篇都齐全，但一旦页码、页眉页脚、分页规则不同，就应分节。

### 默认分区规则表

- 封面：
  - 通常独立分节
  - 通常不显示页码
  - 通常不显示页眉页脚
  - 主标题通常 `18pt-24pt`
  - 不使用首行缩进
  - 一般不处理图表和公式
- 摘要：
  - 通常单独分节，或与目录分区处理
  - 通常属于前置部分
  - 可不编号，或使用罗马页码
  - “摘要”标题通常 `14pt-16pt`
  - 摘要正文通常首行缩进 `2` 字符
  - 公式若出现，通常先不编号
- 目录：
  - 通常单独分节
  - 通常沿用前置页码体系
  - “目录”标题通常 `14pt-16pt`
  - 目录项使用 Tab 或点线引导对齐页码
  - 不使用正文首行缩进
- 正文：
  - 通常单独分节
  - 通常从阿拉伯数字 `1` 开始重新编号
  - 通常显示页眉与页脚页码
  - 一级标题通常 `14pt-16pt`
  - 二级标题通常 `13pt-14pt`
  - 三级标题通常 `12pt-13pt`
  - 正文通常 `12pt`
  - 正文通常首行缩进 `2` 字符
  - 图注表注通常 `10.5pt-12pt`
  - 独立公式是否编号由 LLM 判断，不默认强制
- 参考文献：
  - 通常单独起页，必要时单独分节
  - 通常延续正文页码
  - 标题通常按一级标题处理
  - 条目通常使用悬挂缩进
  - 一般不使用首行缩进
  - 一般不新建公式编号体系
- 附录：
  - 通常单独起页，格式变化大时可单独分节
  - 通常延续正文页码
  - 标题通常按一级标题处理
  - 首行缩进按附录内容类型决定
  - 图表宽度可比正文略宽
  - 公式编号仍由 LLM 按语义判断

### 1. 封面

页面与分节：

- 封面通常独立成节。
- 封面之后通常与摘要或目录分开。

页眉页脚：

- 通常无页眉。
- 通常无页脚。
- 通常不显示页码。

字段级规范：

- 主标题：居中，最大字号区间可按 `18pt-24pt` 处理，通常加粗。
- 副标题：若存在，居中，字号低于主标题一级。
- 学校、学院、专业、作者、导师、日期：
  - 可采用居中块布局，或左右标签加内容布局。
  - 若模板偏正式，可使用左标签加 Tab 对齐内容。
  - 不使用首行缩进。
- 校徽或 logo：居中，尺寸适中，不压主标题。

图片与表格：

- 封面一般不使用数据表格。

公式：

- 封面通常不应出现公式。

### 2. 摘要

页面与分节：

- 摘要通常单独成节，或与目录分节。

页眉页脚：

- 可无页眉，或使用简短规范页眉。
- 页脚可不编号，或使用前置页码体系。

页码：

- 多数情况下属于前置部分。
- 可不编号，或使用罗马数字。

字段级规范：

- “摘要”标题：通常居中，加粗，字号高于正文一档，建议区间 `14pt-16pt`。
- 摘要正文：
  - 中文字体 `宋体`
  - 英文字体 `Times New Roman`
  - 首行缩进通常 `2` 字符
  - 行距通常与正文一致或略紧
- 关键词：
  - “关键词”标签可加粗
  - 标签后内容连续排列
  - 一般不另起复杂编号

图片与表格：

- 摘要中通常不鼓励出现大量图片表格。

公式：

- 若出现，通常不编号。

### 3. 目录

页面与分节：

- 目录通常单独成节，尤其是与正文页码不同的时候。

页眉页脚：

- 可无页眉，或使用简短规范页眉。
- 页脚通常使用前置页码体系。

页码：

- 可不编号，或沿用前置页码。
- 正文通常从目录之后重新开始阿拉伯数字编号。

字段级规范：

- “目录”标题：通常居中，加粗，字号高于正文一档，建议区间 `14pt-16pt`。
- 目录项：
  - 不使用正文首行缩进
  - 使用 Tab 对齐页码
  - 可使用点线引导
  - 同一级目录项字号保持一致
  - 不同层级可通过缩进区分

图片与表格：

- 目录中不处理普通图片表格。

公式：

- 目录中不处理普通公式。

### 4. 正文

页面与分节：

- 正文通常单独成节。
- 一级章节通常新页开始。

页眉页脚：

- 页眉通常放论文题目缩写、章节名或学校规范文本。
- 页脚通常放页码。

#### 奇偶页眉规范（学位论文常见配置）

学位论文最常见的页眉规则是：

- 奇数页：显示论文题目（整篇固定不变）。
- 偶数页：显示当前一级标题（由 STYLEREF 字段动态取得）。

LLM 在 `documentRules` 与 `headerFooterDecisions` 中必须做齐以下事项：

1. `documentRules.oddEvenDifferent = true`，触发 settings.xml 写入 `<w:evenAndOddHeaders/>`。
2. 在 `headerFooterDecisions` 中显式声明两份 header XML：
   - `type = "default"`：奇数页页眉，直接写入论文题目文本。
   - `type = "even"`：偶数页页眉，包含 STYLEREF 字段。
3. STYLEREF 字段格式（手写到 even header XML 中）：

   ```xml
   <w:fldChar w:fldCharType="begin"/>
   <w:instrText xml:space="preserve"> STYLEREF "Heading 1" \* MERGEFORMAT </w:instrText>
   <w:fldChar w:fldCharType="separate"/>
   <w:t></w:t>
   <w:fldChar w:fldCharType="end"/>
   ```

   - **开关只能是 `\* MERGEFORMAT`，不要使用 `\s`。** `\s` 取段落编号，对手动编号的标题会返回空。
   - `"Heading 1"` 是模板里一级标题的 styleId，不一定等于显示名"标题 1"。如果模板里用的是中文样式 ID，需要把这里替换成实际 styleId。
4. 在 `sectionDecisions` 中，正文节必须用 `headerDecisionKeys`（复数）同时绑定上述两条决策键，缺一不可。

#### 前置部分（封面/摘要/目录）页眉

- 默认建议：**完全不显示页眉**。前置部分若残留模板里的旧文本（例如硬编码的"摘要"），必须显式使用 `action = "clear"` 决策覆盖，不能依赖引擎自动清空。
- 若启用了奇偶页眉，前置每一节都要给 default 和 even 两份页眉同时声明 `clear`，否则偶数页可能回退到正文页眉。

#### 页码（正文从 1 重新编号）

两种常见实现：

- **正文从 1 起、前置不计页码**：前置节不设 PAGE 字段、不设 pgNumType；正文节 `pgNumType.format = "decimal"`、`pgNumType.start = 1`。
- **前置罗马数字 + 正文阿拉伯数字**：前置节 `pgNumType.format = "upperRoman"`、`pgNumType.start = 1`；正文节 `pgNumType.format = "decimal"`、`pgNumType.start = 1`。
- 页码本身必须以 `PAGE` 字段写入 footer XML，不要在段落里直接写普通数字。

页码：

- 通常从阿拉伯数字 `1` 开始。
- 页码位置通常在页脚居中或右侧。

字段级规范：

- 一级标题：
  - 加粗
  - 建议字号区间 `14pt-16pt`
  - 段前通常大于段后
  - 通常分页前置或新页开始
- 二级标题：
  - 加粗
  - 建议字号区间 `13pt-14pt`
  - 段前段后明显小于一级标题
- 三级标题：
  - 可加粗或常规字重
  - 建议字号区间 `12pt-13pt`
- 正文：
  - 中文字体：`宋体`
  - 英文字体：`Times New Roman`
  - 正文字号：通常 `12pt`
  - 首行缩进：通常 `2` 字符
  - 行距：通常 `1.5`

图片：

- 默认居中。
- 宽度建议版心 `60%` 到 `85%`。
- 图注放图下方。
- 多张同类图片优先统一宽度。

表格：

- 默认居中。
- 宽度建议版心 `80%` 到 `100%`。
- 表注放表上方。
- 表头通常加粗。
- 表内文字一般不首行缩进。

图注与表注相对字号：

- 图注、表注通常可比正文小半档到一档。
- 若正文 `12pt`，图注表注可落在 `10.5pt-12pt` 区间。
- 图注表注行距通常不大于正文。

公式：

- 独立公式居中。
- 行内公式不编号。
- 独立公式是否编号由 LLM 判断：
  - 被正文引用、需要正式编号体系时，通常编号。
  - 普通推导中的中间公式可不编号。
- 编号存在时通常右对齐。

### 5. 参考文献

页面与分节：

- 通常单独起节，很多情况下也单独起页。

页眉页脚：

- 可沿用正文页眉页脚，也可按学校规范调整。

页码：

- 通常延续正文页码，不重新编号。

字段级规范：

- “参考文献”标题通常按一级标题处理。
- 条目通常使用悬挂缩进。
- 一般不使用正文式首行缩进。
- 行距可与正文一致，或略紧凑。

图片与表格：

- 通常不作为重点。

公式：

- 一般不处理正式公式编号。

### 6. 附录

页面与分节：

- 通常单独起节。
- 如格式与正文差异较大，可单独分节。

页眉页脚：

- 可沿用正文规则，或使用“附录”相关页眉。

页码：

- 通常延续正文页码。

字段级规范：

- “附录”标题通常按一级标题处理。
- 附录正文按内容类型决定是否首行缩进。

图片与表格：

- 若附录主要是图表材料，可适当放宽图片和表格宽度。

公式：

- 若附录含推导，可继续沿用公式规则。
- 是否编号仍由 LLM 判断，不默认强制。

## 模板二：`experiment-report`

适用：

- 实验报告
- 实训报告
- 课程作业型技术文档

### 结构规范

- 通常比论文短，分节较少。
- 页码或页面方向变化时再分节。

### 页眉页脚

- 一般可简化。
- 页脚常只放页码。

### 页码

- 多数情况全篇统一阿拉伯数字。

### 标题

- 标题层级清晰即可。
- 一级、二级标题差异明显。

### 正文

- 中文字体：`宋体`
- 英文字体：`Times New Roman`
- 首行缩进可保留 `2` 字符，也可不缩进，按模板判断。
- 行距通常 `1.25` 到 `1.5`

### 图片

- 流程图、实验照片、结果截图通常居中。
- 图注放图下方。

### 表格

- 数据表优先保证可读性。
- 表注放表上方。
- 数值列居中或右对齐。
- 表头通常加粗。

### 公式

- 独立公式居中。
- 行内公式不编号。
- 是否编号由 LLM 判断。

## 模板三：`company-report`

适用：

- 正式汇报材料
- 商务报告
- 内部方案文档

### 结构规范

- 一般不做复杂分节。
- 封面、目录、正文、附录规则变化时再分节。

### 页眉页脚

- 页眉可放公司名、文档名、日期、部门信息。
- 页脚可放页码或保密级别。

### 页码

- 一般全篇统一阿拉伯数字。

### 标题

- 标题层级要强。
- 一级标题通常更大或更粗。

### 正文

- 中文字体：`宋体` 或模板商务字体
- 英文字体：`Times New Roman` 或 `Arial`
- 行距通常 `1.25` 到 `1.5`
- 首行缩进视模板而定

### 图片

- 图片、流程图通常居中。
- 宽图可更宽，但不能溢出版心。
- 图注可保留，也可适当简化。

### 表格

- 表头通常加粗并有清晰边框。
- 数值表注意列宽均衡。
- 表格可满宽，但不应拥挤。

### 公式

- 只有确有公式内容时处理。
- 是否编号由 LLM 按上下文判断。

## 模板四：`meeting-minutes`

适用：

- 会议纪要
- 沟通记录
- 访谈纪要

### 结构规范

- 通常不需要复杂分节。
- 除封面或附件外，多数情况全文一节。

### 页眉页脚

- 页眉可放“会议纪要”或会议主题。
- 页脚通常只放页码。

### 页码

- 通常全篇统一阿拉伯数字。

### 标题

- 主标题清晰即可。
- 小标题多用于模块化信息。

### 正文

- 多数纪要更适合块状信息排列。
- 列表优先于长段落首行缩进。

### 图片

- 若有会议截图或附图，居中并加图注。

### 表格

- 用于名单、分工、计划时要简洁清晰。

### 公式

- 基本不是重点。
- 若出现，按实际语义处理。

## 模板五：`general-formal-doc`

适用：

- 无法准确归类时的默认模板
- 一般正式文档

### 结构规范

- 默认尽量少分节。
- 只有页码、页眉页脚、方向、边距规则变化时才分节。

### 页眉页脚

- 保留简单页眉页脚或模板默认值。

### 页码

- 如无特别要求，正文统一阿拉伯数字页码。

### 标题

- 一级到三级标题与正文必须明显区分。

### 正文

- 中文字体：`宋体`
- 英文字体：`Times New Roman`
- 正文字号：`12pt`
- 首行缩进：`2` 字符
- 行距：`1.5`

### 图片

- 居中
- 不超过版心
- 图注在下方

### 表格

- 居中
- 不超过版心
- 表注在上方
- 表头可加粗
- 默认按三线表处理：顶线、表头下横线、底线
- 除非模板或用户明确要求，不默认使用满框边线

### 公式

- 独立公式居中。
- 行内公式跟随正文。
- 是否编号由 LLM 按上下文判断。

## 图片设置规则

按以下顺序判断：

1. 它是普通图片、流程图、截图，还是公式截图。
2. 是否需要图注。
3. 图注位置，默认在图下方。
4. 是否需要统一宽度。

推荐规则：

- 普通说明性图片：居中，宽度 `60%` 到 `80%`
- 宽图或流程图：`85%` 到 `100%`
- 多张同类图片：优先统一宽度
- 图片前后留适度段距

## 表格设置规则

按以下顺序判断：

1. 它是数据表、说明表、清单表还是名单表。
2. 是否需要表注。
3. 表注位置，默认在表上方。
4. 是否需要统一表头样式。
5. 列宽、对齐方式、边框强度。

推荐规则：

- 数据表：表头加粗，数值列居中或右对齐
- 清单表：文字列左对齐
- 名单表：结构简洁，不过重边框
- 表格整体宽度：`80%` 到 `100%`

## 图注与表注规则

- 图注默认放图下方。
- 表注默认放表上方。
- 图注表注通常不首行缩进。
- 图注表注字号通常略小于正文或与正文一致但更紧凑。

## 公式设置规则

- 原生 Word 公式优先按公式对象处理。
- 独立公式通常居中。
- 行内公式不单独断行。
- 是否编号不能写死，必须由 LLM 判断。

优先编号的情况：

- 论文正文中的正式公式
- 后文会以“式(1)”“公式(2)”引用的公式
- 需要形成连续编号体系的核心公式

通常不编号的情况：

- 行内公式
- 临时计算过程中的短公式
- 没有被引用、没有编号必要的过渡公式

编号存在时，通常右对齐。

## 模板选择原则

- 用户明确指定模板时，优先使用指定模板。
- 用户上传模板文档时，优先按模板文档规则处理。
- 用户只给自然语言要求时，按最接近模板映射。
- 无法准确匹配时，回退到 `general-formal-doc`。
