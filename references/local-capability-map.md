# Local Capability Map

## 目标

该文件定义 `docx-smart-format` skill 在本地默认拥有的 Word 文档能力边界。

重点不是描述外部依赖，而是明确：

- skill 本地能读到什么
- skill 本地能改什么
- LLM 什么时候该调用哪类能力

## 本地化原则

- 这套 skill 的模板规则、schema、能力说明全部放在本目录中。
- skill 使用时不依赖外部样式服务、外部模板库、远程规则引擎。
- skill 使用时也不要求再进入原项目目录重新检索能力定义。
- 如果原项目已经验证过某些 docx 能力，应在这里固化成可复用的本地规则。

## 本地可读能力

LLM 可以假定本地能力层能读取以下对象：

- 文档级对象：
  - 页面大小
  - 页面方向
  - 页边距
  - 节数量与节边界
  - 页眉页脚引用关系
  - 页码起始值与编号格式
- 段落级对象：
  - 标题层级
  - 对齐方式
  - 首行缩进
  - 悬挂缩进
  - 左右缩进
  - 行距
  - 段前段后
  - 分页前
  - 与下段同页
- 行内对象：
  - 中文字体
  - 英文字体
  - 字号
  - 加粗
  - 斜体
  - 下划线
  - 字体颜色
  - 上标
  - 下标
  - Tab
- 资源对象：
  - 图片
  - 表格
  - 原生 Word 公式
  - 图注
  - 表注

## 本地可写能力

LLM 可以假定本地能力层能写回以下对象：

- 文档级写回：
  - 新建或复用节
  - 设置页面方向与页边距
  - 设置首页不同
  - 设置奇偶页不同
  - 插入页码字段
  - 复用或重建页眉页脚
- 段落级写回：
  - 标题样式映射
  - 正文样式映射
  - 图注表注样式映射
  - 参考文献样式映射
  - 段前段后
  - 行距
  - 缩进
  - 分页控制
- 行内写回：
  - 中文字体与英文字体分别设置
  - 加粗、斜体、下划线
  - 上标、下标
  - 字号与颜色
  - 保留或插入 Tab
- 资源级写回：
  - 图片宽度、对齐、前后间距、题注位置
  - 表格宽度、表头、边框、单元格对齐、题注位置
  - 原生公式对齐、前后间距、是否编号、编号位置

## LLM 调用边界

LLM 负责做判断，不负责做底层 XML 拼接。

LLM 应输出：

- 文档属于哪种模板
- 每个块属于什么语义角色
- 每个 run 是否需要保留或改写样式
- 每个图片、表格、公式对象应该如何格式化
- 哪些地方需要分节、页眉页脚、页码字段

LLM 不应输出：

- Word XML 片段
- 模糊描述，例如“稍微大一点”“看起来正式一些”
- 没有字段落点的自然语言格式要求

## 调用顺序

建议按以下顺序判断：

1. 先定文档类型与模板。
2. 再定分区与分节。
3. 再定标题、正文、参考文献等块级角色。
4. 再定行内 run 的语义样式。
5. 最后定图片、表格、公式等对象格式。

## 典型调用模式

### 分节

当封面、摘要、目录、正文、附录的页码或页眉页脚规则不同，调用分节能力。

### 页眉页脚与页码

当正文需要正式编号、前置部分需要不同编号体系、首页不显示页码时，调用页眉页脚和页码字段能力。

### 行内强调

当术语、变量、化学式、单位指数、书名式强调出现时，调用加粗、斜体、上标、下标等 run 能力。

### 图表

当对象是图片或表格时，先判断它的内容角色，再决定宽度、对齐、题注位置和前后间距。

### 公式

当对象是原生 Word 公式时，先判断行内还是独立，再判断是否需要编号。
只有明确被引用或模板明确要求时，才倾向编号。

## 与模板的关系

- 模板给出默认规则。
- LLM 结合模板和源文档内容做最终决策。
- 若模板和源文档冲突，以模板为主，以语义合理性为约束。
- 若没有模板，回退到 `general-formal-doc`。
